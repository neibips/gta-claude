# Техническое задание для нейросети

## Проект: GTA6 AI

Создать браузерную sandbox-игру в стиле GTA на BabylonJS.

Игра должна запускаться в браузере и содержать заранее созданную постоянную карту города, игрока от третьего лица, NPC, полицию, оружие, транспорт, систему розыска, HUD и автотесты.

---

## Стек

- Vite
- TypeScript
- BabylonJS 9
- BabylonJS Havok Physics
- BabylonJS GUI
- Playwright
- Vitest
- Context7 MCP с `/babylonjs/babylon.js`

При написании BabylonJS-кода использовать Context7 для проверки актуального BabylonJS API.

---

# 1. Главный принцип по карте

Карта не должна генерироваться во время запуска игры.

Карта не должна генерироваться при первом запуске.

Карта должна быть заранее создана нейросетью как готовый файл:

```text
assets/maps/city-map.json
```

Игра всегда должна загружать карту только из этого файла.

Если файла карты нет, игра должна показать понятную ошибку:

```text
Map file not found: assets/maps/city-map.json
```

Автоматически создавать новую карту во время запуска игры запрещено.

Город должен быть постоянным и одинаковым при каждом запуске.

# 4. Файл карты

## 4.1 Обязательное требование

Нужно создать готовый файл карты:

```text
assets/maps/city-map.json
```

Этот файл должен быть частью проекта.

Игра должна только читать этот файл и строить по нему сцену.

В игре не должно быть runtime-генерации города.

Разрешено использовать `MapBuilder.ts` только как построитель 3D-сцены из готовых данных карты, а не как генератор новой карты.

Правильная логика:

```text
загрузка игры
→ загрузка assets/maps/city-map.json
→ валидация карты
→ построение сцены по данным из файла
→ запуск игрового процесса
```

Неправильная логика:

```text
загрузка игры
→ генерация случайной карты
→ сохранение карты
→ запуск игры
```

## 4.2 Размер карты

Карта должна быть размером:

```text
200 × 200 единиц
```

Карта должна быть детально продумана заранее.

Город не должен ощущаться пустым.

# 5. Формат assets/maps/city-map.json

Создать JSON-файл со следующей структурой:

```ts
type CityMapFile = {
  version: number;
  name: string;
  seed: string;

  size: {
    width: number;
    height: number;
  };

  roads: SavedRoad[];
  sidewalks: SavedSidewalk[];
  buildings: SavedBuilding[];
  trees: SavedTree[];
  decorations: SavedDecoration[];

  npcWaypoints: SavedWaypoint[];
  trafficWaypoints: SavedWaypoint[];
  coverPoints: SavedCoverPoint[];

  spawnPoints: {
    player: Vector3Like;
    npc: Vector3Like[];
    police: Vector3Like[];
    cars: Vector3Like[];
  };
};

type Vector3Like = {
  x: number;
  y: number;
  z: number;
};

type SavedRoad = {
  id: string;
  type: "perimeter" | "inner" | "intersection";
  position: Vector3Like;
  size: {
    width: number;
    length: number;
  };
  rotationY: number;
  materialId: string;
};

type SavedSidewalk = {
  id: string;
  position: Vector3Like;
  size: {
    width: number;
    length: number;
  };
  rotationY: number;
};

type SavedBuilding = {
  id: string;
  type: "residential" | "office" | "shop" | "parking" | "police_station" | "warehouse";
  position: Vector3Like;
  size: {
    width: number;
    depth: number;
    height: number;
  };
  floors: number;
  rotationY: number;
  materialId: string;
  windowPattern: string;
};

type SavedTree = {
  id: string;
  position: Vector3Like;
  scale: number;
  type: "billboard" | "low_poly";
};

type SavedDecoration = {
  id: string;
  kind: "lamp" | "bench" | "trash_bin" | "sign" | "fence" | "hydrant" | "flower_bed" | "curb";
  position: Vector3Like;
  rotationY: number;
  scale: number;
};

type SavedWaypoint = {
  id: string;
  position: Vector3Like;
  links: string[];
  type: "npc" | "traffic";
};

type SavedCoverPoint = {
  id: string;
  position: Vector3Like;
  direction: Vector3Like;
  type: "building_corner" | "car" | "fence" | "decoration" | "police_station";
};
```

# 6. Содержание карты

## 6.1 Общая планировка

Файл `city-map.json` должен описывать небольшой городской район.

Обязательные зоны:

1. Внешняя дорога по периметру карты
2. Внутренние дороги
3. Один или несколько перекрёстков
4. Дворовая зона
5. Парковая зона
6. Парковка
7. Пешеходные тротуары
8. Полицейский участок
9. Жилые здания
10. Офисные здания

## 6.2 Дороги

Дорога по периметру:

- ширина 18 единиц
- подходит для движения двух машин
- соединена с внутренними дорогами
- имеет повороты на углах

Внутренние дороги:

- ширина 12–16 единиц
- связывают зоны карты
- имеют маршруты для трафика

Для дорог использовать материал:

- тёмный асфальт
- дорожная разметка
- бордюры

## 6.3 Тротуары

Тротуары:

- ширина 5 единиц
- расположены вдоль дорог
- используются NPC

Waypoint-сеть:

- waypoint примерно каждые 8 единиц
- waypoints связаны между собой
- NPC ходят только по pedestrian waypoints
- транспорт не использует pedestrian waypoints

## 6.4 Здания

В файле карты должно быть 6–12 зданий.

Высота:

- от 2 до 12 этажей
- 1 этаж = 4 единицы

Типы зданий:

- жилой дом
- офисное здание
- магазин
- парковка
- полицейский участок
- склад или техническое здание

Требования:

- здания не пересекаются с дорогами
- между зданиями есть проходы
- здания имеют разные размеры
- здания имеют разную высоту
- здания визуально отличаются
- рядом со зданиями есть cover points для полиции

Материалы:

- PBR
- разные цвета фасадов
- emissive окна
- roughness / metallic параметры

## 6.5 Полицейский участок

На карте обязательно должен быть полицейский участок.

Требования:

- отдельное здание
- рядом парковка
- рядом spawn point полиции
- рядом несколько cover points
- визуально отличимое здание

## 6.6 Парковая зона

Создать небольшую парковую зону во дворе или в центре карты.

В парке:

- деревья
- скамейки
- фонари
- пешеходные дорожки
- NPC waypoints
- cover points

## 6.7 Деревья

Создать 8–12 деревьев.

Размещение:

- часть по периметру карты
- часть во дворе
- часть в парковой зоне

Реализация:

- BillboardMode или low poly tree mesh
- простые коллизии
- тень
- не блокировать основные маршруты NPC и транспорта

## 6.8 Декор

Добавить в файл карты:

- фонари
- скамейки
- мусорки
- заборы
- гидранты
- клумбы

Декор не должен ломать навигацию NPC и транспорта.

# 7. Загрузка карты в игре

Создать `MapLoader.ts`.

Он должен:

- загрузить `assets/maps/city-map.json`
- проверить наличие файла
- распарсить JSON
- проверить `version`
- проверить размер карты
- проверить обязательные массивы
- передать данные в `MapBuilder`

Создать `MapValidator.ts`.

Он должен проверять:

- карта имеет размер 200×200
- есть дороги
- есть тротуары
- есть здания
- есть police_station
- есть spawn point игрока
- есть spawn points NPC
- есть spawn points полиции
- есть spawn points машин
- есть NPC waypoints
- есть traffic waypoints
- есть cover points
- количество деревьев от 8 до 12
- здания не имеют очевидных пересечений с дорогами
- waypoint-ссылки указывают на существующие waypoints

Создать `MapBuilder.ts`.

Он должен:

- построить ground
- построить дороги
- построить тротуары
- построить здания
- построить деревья
- построить декор
- создать физические коллизии
- создать визуальные материалы
- создать waypoint-графы
- создать cover points
- создать spawn points

`MapBuilder.ts` не должен создавать новую карту.  
Он только строит сцену по данным из `city-map.json`.

# 8. Физика и материалы карты

Для объектов карты использовать физические тела:

- здания — статичные коллизии
- дороги — ground collision
- тротуары — ground collision
- деревья — простые impostor-коллизии
- декор — простые impostor-коллизии

Материалы:

- дороги — асфальт
- тротуары — бетон
- здания — PBR
- окна — emissive
- деревья — простые материалы или billboard texture
- декор — простые материалы

Не создавать слишком тяжёлую физику для статичных объектов.

# 10. Игрок

## 10.1 Модель

Модель игрока загружать из:

```text
assets/player/
```

Использовать существующие ассеты из этой папки.

## 10.2 Камера

Камера:

- ArcRotateCamera
- вид от третьего лица
- радиус 8
- высота 3
- плавное следование за игроком
- движение не должно быть инвертировано

## 10.3 Физика игрока

Использовать CapsuleImpostor:

- высота 1.8
- радиус 0.3

Игрок должен:

- ходить
- бегать
- поворачиваться
- сталкиваться со зданиями
- сталкиваться с машинами
- не проваливаться под карту

## 10.4 Управление

Клавиши:

- W — вперёд
- S — назад
- A — влево
- D — вправо
- Shift — бег
- E — взаимодействие / посадка в авто
- F — выход из авто
- 1–4 — выбор оружия
- ЛКМ — атака / выстрел

Скорость:

- шаг = 4
- бег = 9

Поворот:

- Quaternion.Slerp
- направление движения от камеры
- не инвертировать направление

## 10.5 Здоровье и смерть

HP игрока = 100.

При получении урона:

- уменьшать HP
- проигрывать hit_reaction
- обновлять HUD

При смерти:

- проиграть death_fall
- сбросить розыск до 0
- отключить управление
- через 3 секунды возродить игрока
- восстановить HP до 100
- вернуть управление

# 13. NPC

## 13.1 Модели

NPC загружать из:

```text
assets/npc/
```

Выбирать модель случайно из доступных ассетов.

## 13.2 Количество

Максимум 20 NPC на карте.

## 13.3 Спавн

Использовать spawn points из `assets/maps/city-map.json`.

NPC спавнятся каждые 8 секунд, если лимит не превышен.

## 13.4 Движение

NPC ходят по waypoint-сети тротуаров из `city-map.json`.

NPC должны:

- выбирать случайный waypoint
- идти к нему
- после достижения выбирать следующий
- не идти по дороге без необходимости
- обходить препятствия простым steering-поведением

## 13.5 FSM

Состояния:

- WALKING
- FLEEING
- DEAD

## 13.6 Реакция на выстрел

Если NPC слышит выстрел ближе 25 единиц:

- состояние FLEEING
- скорость ×2.5
- анимация run
- движение от источника опасности

## 13.7 Смерть NPC

При смерти:

- включить ragdoll
- создать PhysicsImpostor на основные кости
- применить импульс от попадания
- повысить розыск
- оставить тело на земле

Импульсы:

- AK-47 = 8
- RPG = сферическая волна 30

Если трупов на карте `>= 10`, удалить самый старый через `dispose()`.

# 14. Система розыска

Розыск: от 0 до 5 звёзд.

Правила:

- убийство NPC: +1 звезда
- убийство полицейского: +2 звезды
- 30 секунд без нарушений: −1 звезда
- смерть игрока: 0 звёзд

Количество полиции:

- 1 звезда: 1 полицейский
- 2 звезды: 2 полицейских
- 3 звезды: 4 полицейских
- 4 звезды: 6 полицейских
- 5 звёзд: 8 полицейских

WantedSystem должен:

- хранить текущий уровень розыска
- уведомлять HUD
- управлять спавном полиции
- снижать розыск при отсутствии нарушений
- сбрасывать розыск при смерти игрока

# 15. Полиция

## 15.1 Модели

Модели загружать из:

```text
assets/policeman/
```

Выбирать случайную модель из доступных ассетов.

## 15.2 Параметры

HP полицейского = 80.

Оружие выбирается случайно из разрешённого набора.

## 15.3 FSM полиции

Состояния:

- PATROL
- SEARCH
- CHASE
- TAKE_COVER
- FLANK
- ATTACK
- RETREAT
- DEAD

## 15.4 Тактический AI

Полиция должна вести себя тактически.

Требования:

- использовать укрытия из `city-map.json`
- искать ближайшие cover points
- занимать позиции вокруг игрока
- обходить игрока с флангов
- создавать перекрёстный огонь
- не собираться в одной точке
- не стрелять через союзников
- не стрелять через стены
- не стрелять сидя или лёжа
- стрелять только стоя
- выходить из укрытия перед выстрелом
- после выстрела возвращаться в укрытие
- менять позицию каждые 3 секунды
- при низком HP отходить назад

## 15.5 Атака

Правила:

- стрелять каждые 1.5 секунды
- перед выстрелом проверять line of sight
- стрелять только если полицейский стоит
- при отсутствии line of sight переходить в CHASE или FLANK
- если рядом есть cover point, переходить в TAKE_COVER

## 15.6 Укрытия

Cover points берутся из `assets/maps/city-map.json`.

Укрытия должны быть:

- у углов зданий
- у машин
- у заборов
- у декоративных объектов
- рядом с полицейским участком

Для cover point хранить:

- позицию
- направление
- тип укрытия

## 15.7 Смерть полиции

При смерти:

- повысить розыск на +2
- включить ragdoll
- применить импульс
- удалить самый старый труп, если тел слишком много

# 16. Транспорт

## 16.1 Модели

Машины загружать из:

```text
assets/car/
```

Выбирать случайную модель из доступных ассетов.

## 16.2 Трафик

На карте одновременно 3–5 машин трафика.

Машины должны:

- ездить по road waypoints из `city-map.json`
- останавливаться перед препятствиями
- не ехать по тротуарам
- поворачивать на перекрёстках
- не вылетать с дороги

RayCast:

- 10 единиц вперёд
- остановка перед игроком
- остановка перед NPC
- остановка перед другой машиной

## 16.3 Посадка в машину

Если игрок рядом с машиной ближе 3 единиц и нажимает E:

- игрок садится в машину
- управление игроком отключается
- камера привязывается к машине
- машина становится управляемой

## 16.4 Выход из машины

Клавиша F:

- остановить машину или снизить скорость
- переместить игрока рядом с дверью
- включить управление игроком
- вернуть камеру к игроку

## 16.5 Физика машины

Использовать Havok VehicleImpostor или raycast wheels.

Можно использовать vechile-example.ts для создания управления машины

Управление:

- W — газ
- S — тормоз / назад
- A — поворот влево
- D — поворот вправо

Параметры:

- максимальная скорость 40
- дрифт за счёт уменьшенного бокового сцепления
- машина имеет массу
- машина сталкивается с объектами

## 16.6 Наезд на NPC

При столкновении машины с NPC:

- импульс = скорость × 2.5
- если скорость > 20, урон смертельный
- повысить розыск
- включить ragdoll NPC

# 21. Критические требования

1. Карта должна быть заранее создана в файле `assets/maps/city-map.json`.
2. Игра не должна генерировать карту при запуске.
3. Игра не должна генерировать карту при первом запуске.
4. Игра должна только загружать готовую карту из файла.
5. Если файла карты нет, показать ошибку.
6. Размер карты 200×200.
7. В файле карты должны быть дороги, здания, тротуары, деревья, декор, waypoints, cover points и spawn points.
8. Движение игрока не должно быть инвертировано.
9. Оружие должно крепиться плотно к кости RightHand.
10. Оружие использует offset из WeaponConfig.json.
11. NPC ходят по waypoint-сети тротуаров.
12. Транспорт ездит по road waypoints.
13. Полиция использует тактический AI.
14. Полиция стреляет только стоя.
15. Полиция использует укрытия.
16. Полиция проверяет line of sight.
17. Трупы удаляются при количестве >= 10.
18. Использовать Context7 для актуального BabylonJS API.
19. Playwright должен проверить запуск игры и загрузку карты из файла.
20. Vitest должен проверить основные системы.
21. Код должен быть production-ready.
22. Все ассеты брать только из существующих папок assets/.
23. Ошибки загрузки ассетов должны обрабатываться.
